# eip712dsa

An **EIP-712 Domain Separator Auditor** for the [Pharos Network](https://pharos.xyz). Give it a contract address on Pharos Atlantic Testnet or Pacific Ocean Mainnet, and it audits the deployed bytecode for EIP-712 correctness — checks the type-hash literal, verifies the chain-id is fetched via the `chainid()` opcode (not hardcoded), confirms the `verifyingContract` is `address(this)`, and flags the common pitfalls (fixed salt, missing version, hardcoded chainId).

Ships as a [Pharos Agent Center](https://www.pharos.xyz/agent-center) skill — drop it into Claude / Codex / OpenClaw and the agent can audit any Pharos contract's EIP-712 implementation on demand.

## What it checks

| # | Check | What it looks for in bytecode | Severity |
|---|---|---|---:|
| 1 | `EIP712Domain` type-hash literal present | the 82-byte string `EIP712Domain(string name,string version,uint256 chainId,address verifyingContract)` | 100 |
| 2 | `name` field present | a string literal whose keccak256 matches a known token name (e.g. `USD Coin`, `Pharos Token`) | 50 (info) |
| 3 | `version` field present | a string literal whose keccak256 matches a known version string (e.g. `1`, `2`, `v1`) | 30 (info) |
| 4 | `chainId` is fetched via `chainid()` opcode, not hardcoded | the bytecode contains the `0x46` CHAINID opcode near the DOMAIN_SEPARATOR computation | 95 |
| 5 | `verifyingContract` is `address(this)` | the bytecode contains the `0x30` ADDRESS opcode near the DOMAIN_SEPARATOR computation | 90 |
| 6 | `DOMAIN_SEPARATOR()` is a public view | the bytecode contains the function selector `0x3644e515` (DOMAIN_SEPARATOR()) | 100 |
| 7 | No fixed-salt trap | the bytecode does NOT contain a fixed `bytes32 salt` (would be replayable across deployers) | 80 |
| 8 | Domain separator includes a salt (optional) | the bytecode contains the `bytes32 salt` field — info only | 0 (info) |

Each check has one of three verdicts: **PASS**, **FAIL**, or **NOT_FOUND** (the bytecode doesn't implement EIP-712 at all).

## Install

```bash
# Option A — git clone
git clone https://github.com/Tunde0058/EIP-712DSA.git
cd EIP-712DSA
chmod +x scripts/audit.sh scripts/audit_demo.sh
pip install web3            # only for the Python version

# Option B — one-line via OpenClaw
npx skills add https://github.com/Tunde0058/EIP-712DSA

# Or: install as a Pharos Agent Center / Claude Code / Codex / OpenClaw skill
mkdir -p ~/.pharos/skills
cp -r . ~/.pharos/skills/eip712dsa
```

## Quick start

### Zero-dependency (bash + curl only)

```bash
# Default: Markdown report, mainnet
bash scripts/audit.sh 0xYOUR_CONTRACT --network mainnet

# Machine-readable JSON
bash scripts/audit.sh 0xYOUR_CONTRACT --network mainnet --format json

# Strict mode: exit 1 on any FAIL (useful in CI)
bash scripts/audit.sh 0xYOUR_CONTRACT --network mainnet --strict
```

### Python (richer output)

```bash
pip install web3
python3 scripts/audit.py 0xYOUR_CONTRACT --network mainnet --format md
```

### Run the demo

```bash
bash scripts/audit_demo.sh
```

This audits a real public mainnet contract and prints a sample report.

### Verify the install

```bash
bash scripts/audit.sh --help
python3 scripts/audit.py --help
python3 tests/test_audit.py
```

## Honest scope

eip712dsa is a **starting point**, not a verdict. It catches the obvious EIP-712 pitfalls but does NOT substitute for a full audit firm. Use it to triage contracts quickly, then hand off to a real audit for the ones that flag.
